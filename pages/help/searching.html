Searching

Search queries for the stash use a simple syntax that will be explained in this guide. It's designed such that you usually just need to enter what you want to see.

For example, to see pickaxes of a specific material, enter iron pickaxe or diamond pickaxe.


The last example above uses two conditions, diamond and pickaxe. These are file tags, which describe the files they apply to. Below is a list of all types of conditions.

Conditions

Tags

To search for files with a specific tag, just enter the tag name. It's as simple as that. Example: armor

To search for files with a specific explicit tag*, put quotes around the tag name. Example: "entity"

* See Implicit Tags below

Contributor

To search for files by a specific contributor, put @ before the contributor's name. Example: @Someone

If the name has any spaces in it, put quotes around the name. Example: @"Someone Else"

File ID

To search for a file with a specific ID, just enter the ID number. Example: 17590

To search for files with an ID in a range, enter the lower and upper limits of the range, separated by -. Example: 100-400

To search for files with an ID relative to a value, put one of the following operators before the value: < > <= >=. Example: <2000

Texture Size

To search for textures with a width relative to a value, enter w and one of the following operators before the value: = < > <= >=. Example: w>=64

To search for textures with a height relative to a value, enter h and one of the following operators before the value: = < > <= >=. Example: h=32

Texture Aspect Ratio

To search for textures with a specific aspect ratio, enter X:Y, where X and Y are the relative width and height respectively. Example: 2:1 - Matches textures where the width is two times the height.

To search for textures with an aspect ratio relative to a value, one of the following operators before the value: < > <= >=. Example: <1:1 - Matches textures that have a height greater than their width.

File Date

To search for files posted relative to a date, enter <OP>{<DATE>}, where OP is one of the following operators: < > <= >=, and DATE is any date string parsable by JavaScript's Date.parse() method. Example: >{Jan 1, 2018}

Number of Tags

To search for files with an explicit tag-count relative to a value, enter t and one of the following operators before the value: = < > <= >=. Example: t>=1

Sources

To search for files that are derived from another on the stash, enter source= before the ID of the source file. Example: source=7311

To search for files that are derived from something else, enter source= before the name of the source. Example: source=ExampleSource

If the source name has any spaces in it, put quotes around the name. Example: source="Dokucraft Light"

Number of Sources

To search for files with a source-count relative to a value, enter sources and one of the following operators before the value: = < > <= >=. Example: sources>=1

Logical Operators

If you want to use multiple conditions or negate a condition, you have to use logical operators. It's quite simple, so don't panic just yet.

AND

In a case where you want two or more conditions to all apply to the results, just separate them with spaces. The order does not matter. Example: armor w=32 @Someone

OR

In a case where you want any of two or more conditions to apply to the results, separate them with |. The order does not matter. Example: gold|iron|diamond

NOT

In a case where you want a condition to not apply to the results, put ! before the condition. Example: !sword

Operator Precedence & Groups

The order of operations is: groups > NOT > OR > AND. That means iron|gold pickaxe will find files matching either ironpickaxe or goldpickaxe.

You can use parentheses to create groups. Example: iron|(gold pickaxe) - This will match files tagged iron orgoldpickaxe.

Implicit Tags

There is a bit more to tags than what appears at first glance. Tags you can see on a file are explicit tags.
Some tags imply that one or more other tags should be present. In such a case, the stash will act as if the other tag also applies to the file, even if it's not there. These are implicit tags.


For example, an ironpickaxe is also a metal tool. If a file has the iron tag, or any other tag that is a metal, the metal tag will also apply to the file.
Similarly, a sword is also a melee-weapon, which is also a weapon.
A bow is a weapon, but not a melee-weapon.


This is to make searching for more vague tags possible without cluttering the tags list for each file. A creeper texture doesn't need creepermonstermobentity, just creeper.


If you want to find only files with a vague explicit tag, put quotes around the tag name as explained in the Conditions section. Example: weapon will also show all files tagged sword, but "weapon" will only show files with the weapon tag.
