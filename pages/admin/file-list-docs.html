Status Page File Lists

A list ID is the ID of a Google Sheet. The sheet's sharing setting must be set to "Anyone who has the link can view/edit"

The ID for a sheet is in the URL. Example:https://docs.google.com/spreadsheets/u/0/d/185N6QJIcvOPZ1AxSquRsnJQbPtf-oQXSQm97XLjBErg/view

The sheet name, which usually is Sheet1, must be changed to Files. The first three rows of the sheet follow this format:

CellValueNote
A1File List TitleDisplayed at the top of the status page.
A2Patch DescriptionUsed as the pack.mcmeta description in the patch. ${packTitle} will be replaced by the title of the pack being downloaded.
A3-Unused.
B1Sheet StatusIf set to wip, a message about the list being worked on is displayed on the status page instead of the file list.
B2Resource Pack Format NumberUsed in pack.mcmeta.
B3Download Link VisibilityOptional. If set to FALSE, the download links for the patch will be hidden. Defaults to the links being visible.
C1Patch ZIP FilenameThe name format of the patch zip. ${pack} in the filename will be replaced by the short name of the pack being downloaded.
C2Status TextOptional. Text that displays below the progress bar(s) on the status page. Can be used to inform users about what is going on with the pack(s).
C3Root DirectoryPath to prepend to the file names/paths in the patch. Defaults to assets/minecraft/textures/ if left blank. To set it to the root directory of the zip, set it to _root_
D,E,F,...1Short Pack NameUsed as an identifier for the pack and in the table header on the status page.
D,E,F,...2Pack TitleFull title of the pack. Used in the progress section on the status page and optionally in pack.mcmeta.
D,E,F,...3-Unused.

Note that there can be any number of packs - just add more columns.

Rows below the first three each represent a file and follow this format:

ColumnValueNote
ACategoryUsed to group files together on the status page. If set to Hidden, the file will not be displayed on the status page, but it will still be added to the patch.
BDefault TextureOptional. Base64 encoded PNG image or a direct image link. Displayed as a small icon before the filename on the status page.
CFilename/PathThe path to the file in the resource pack. The root directory set in C3 will be prepended to this path, unless the path begins with assets/.
D,E,F,...Stash File IDID of a file on the stash. If the ID is of a model file, the ID will get an extra indicator next to it and it will count as complete on the status page, but it will not automatically be added to the patch. You can use Hidden files to add the files you need from a model to the patch.
